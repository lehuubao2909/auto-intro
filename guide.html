What AutoIntro Does

AutoIntro is a local-first tool that converts any code repository into a professional 45–60 second intro video (.mp4) without uploading your code.

The Process

  1. Analyzes your repo (reads package.json, scans source files)
  2. Detects the UI (colors, fonts, components, logo)
  3. Determines usage patterns (CLI tool? Web app? SDK? API? Mobile? Desktop?)
  4. Builds a narrative (brief + call-to-action)
  5. Directs with AI (Gemini creates a storyboard of animated scenes)
  6. Renders locally (Remotion + Chromium, nothing uploaded)

Result: A visually polished video showcasing your project's core flow — ready for README, social media, product pages, or pitches.

Privacy: Your repo and video never leave your machine. Only code/analysis text is sent to Gemini for story direction. No screenshots, no uploads to any server.

Prerequisites

System Requirements

  • Node.js ≥ 20 — Check with node --version
  • macOS, Linux, or Windows (any OS Node runs on)
  • ~500 MB free disk space (for Chromium + rendering temp files)

API Key

You need a Gemini API key (free tier available). Without it, AutoIntro uses a lower-quality offline fallback.

Get your Gemini API key →

Free tier: Gemini's free tier includes 15 requests per minute and covers most use cases. If you hit limits, wait 30 seconds or upgrade.

Getting a Gemini API Key

1

Visit Google AI Studio

Open https://aistudio.google.com/apikey in your browser.

2

Sign In

Sign in with your Google account (create one if needed).

3

Create API Key

Click "Create API Key" → "Create API key in new project".

Your key appears in the text field (looks like AIza...).

4

Copy & Save

Click the copy icon next to your key. Save it somewhere safe (you'll paste it in a moment).

Security: Treat your API key like a password. Don't commit it to Git. Use a .env file (which should be in .gitignore).

Clone & Install

Option A: Fresh Clone (Recommended for First-Timers)

1

Clone the Repository

git clone https://github.com/lehuubao/auto-intro.git
cd auto-intro
2

Install Dependencies

If you already have AutoIntro cloned elsewhere, just navigate to it:

npm install

1

In the AutoIntro directory

Open a terminal in the auto-intro folder and create a .env file:

cat > .env << 'EOF'

Replace YOUR_KEY_HERE with the key you copied earlier (looks like AIza...).

2

Make sure .env is in .gitignore so your key never gets committed:

cat .gitignore | grep .env

First Run: Web UI

Start AutoIntro

In the AutoIntro directory, run:

npm run dev -- /path/to/your/repo

Replace /path/to/your/repo with the actual path to any code repository you want to create a video for.

Example

# For a repo in your home directory
npm run dev -- ~/my-awesome-project

# Or if AutoIntro is IN the repo you're analyzing
npm run dev -- .

What Happens

  1. AutoIntro starts a local server on a free port (usually http://localhost:4747 or similar)
  2. Your browser opens automatically to the web UI
  3. You're ready to begin the analysis flow

Your terminal shows:

  AutoIntro running:  http://localhost:4747
  Target repo:       /path/to/your/repo
  Director / triage: gemini-3.5-flash / gemini-3.1-flash-lite
  Flow: Analyze → review brief → approve → render  (add --yes to skip the gate)

  (Ctrl+C to stop)
First-time slow? On the first run, Remotion auto-downloads a headless Chromium (~100 MB). Expect 5–10 minutes on first launch. Subsequent runs are 2–5 minutes.

Web UI Walkthrough

Step 1: Click "Analyze Project"

The web UI shows a big button. Click "Analyze Project" to start scanning your repo.

What's happening:

  • Reading package.json and project metadata
  • Detecting usage type (CLI, SDK, web app, API, mobile, desktop)
  • Scanning source files to extract design (colors, fonts, components)
  • Building an inventory of UI surfaces

Time: 30–60 seconds

Step 2: Review the Brief

Once analysis completes, the web UI shows the "Project Brief" — a markdown summary of what the video will tell the story of.

The brief includes:

  • Project Name — what it's called
  • Usage Type — how people use it (e.g., "CLI tool")
  • Core Value — one-sentence hook
  • Key Features — 3–5 bullets
  • Usage Flow — step-by-step of a typical interaction
  • Call-to-Action — "Get Started" / "Docs" / "GitHub" / etc.

Can you edit it? Yes! Click the edit button to refine the narrative. If the tool got something wrong, fix it here. The brief is the "contract" the video will be built from.

Step 3: Click "Approve & Generate"

Once you're happy with the brief, click "Approve & Generate".

What happens next (takes 2–5 minutes):

  1. Directing — Gemini generates a scene-by-scene storyboard (60–90 sec)
  2. Rendering — Remotion renders each scene in Chromium (1–3 min)
  3. Complete — Your video is ready

You'll see live progress on the screen (e.g., "Rendering 45%").

Step 4: Download or Re-Render

Once the video is done:

  • Preview — Watch it in the browser player
  • Download — Get the .mp4 file
  • Re-render — Edit the brief or storyboard and render again (no re-analysis needed)

Where's the video saved? In <repo>/.auto-intro/trailer.mp4 (also shown on the download page).

Headless Mode (--yes flag)

If you want to skip the web UI approval gate and render immediately, use the --yes or -y flag:

npm run dev -- /path/to/repo --yes

Or short form:

npm run dev -- /path/to/repo -y

What Happens

  1. Analyzes the repo (no browser UI)
  2. Auto-approves the brief (uses defaults)
  3. Renders immediately
  4. Outputs the video path and exits

All progress is printed to the terminal (no interactive UI).

Use Cases

  • CI/CD pipelines (auto-generate trailers on release)
  • Batch processing multiple repos
  • Server-side automation
  • When you want to skip the UI entirely

  [direct/ok] Directing storyboard (Gemini)…

Understanding Outputs

After analysis and rendering, AutoIntro creates a .auto-intro/ folder in your target repo with these files:

File Purpose
repo-facts.json Detected project name, usage type, dependencies, estimated complexity
design-profile.json Extracted theme, palette (primary/secondary colors), fonts, logo URL
component-inventory.json Mapped UI components to 42 primitive shapes/animations
brief.json Approved narrative (structure: project name, value, features, flow, CTA)
brief.md Human-readable brief (what you edited in the web UI)
storyboard.json Scene-by-scene video blueprint (scenes, timing, animations) — editable by power users
trailer.mp4 Your final video. Share this!

Can I Edit These?

Yes, for power users:

  • brief.json / brief.md — Edit via the web UI. Re-rendering picks up changes.
  • storyboard.json — Advanced users can hand-edit scene details (timing, positions, animations). Requires careful JSON editing.
  • The rest — Read-only. Regenerate the analysis if needed.
Safe to delete: You can safely delete the .auto-intro/ folder and re-run AutoIntro. It regenerates everything. Useful if you want to start fresh with a new brief.

All Environment Variables

Complete reference for all configuration options. Set them in your .env file or export them before running.

Variable Type Default Description
GEMINI_API_KEY string (required) Your Gemini API key. Get at aistudio.google.com/apikey. Also accepts GOOGLE_API_KEY.
AUTOINTRO_DIRECTOR_MODEL string gemini-3.5-flash Model for storyboard generation. Try gemini-2.0-pro for higher accuracy (slower, higher cost).
AUTOINTRO_TRIAGE_MODEL string gemini-3.1-flash-lite Model for repo analysis. Use lite version for speed/cost.
AUTOINTRO_PORT number 0 (auto) Server port. 0 = auto-pick a free port. Set to 4747 for a fixed port.
AUTOINTRO_FPS number 30 Video frames per second. Increase to 60 for smoother animations (slower render).
AUTOINTRO_WIDTH number 1920 Video width in pixels. Common values: 1920 (1080p), 1280 (720p), 2560 (1440p).
AUTOINTRO_HEIGHT number 1080 Video height in pixels.
AUTOINTRO_THEME string cinematic-dark Rendering theme. Applied post-generation to set mood/tone.
AUTOINTRO_ACCENT hex color #41A3EF Primary brand color (e.g., #FF5733). Used throughout animations.
AUTOINTRO_ACCENT2 hex color #FCCE50 Secondary accent color. Sparingly used for highlights.
AUTOINTRO_MAX_FULL_READ number 10 Max files to fully read during analysis. Higher = more context, slower. Hard cap.
AUTOINTRO_WORKDIR string .auto-intro Output directory name (created in the target repo).

Example: Custom Configuration

Create a .env file with custom settings:

GEMINI_API_KEY=your_key_here
AUTOINTRO_DIRECTOR_MODEL=gemini-2.0-pro
AUTOINTRO_FPS=60
AUTOINTRO_WIDTH=2560
AUTOINTRO_HEIGHT=1440
AUTOINTRO_ACCENT=#FF5733

Then run as usual:

npm run dev -- /path/to/repo

Swapping LLM Models

Why Change Models?

  • Higher accuracy: Use gemini-2.0-pro instead of gemini-3.5-flash
  • Lower cost: Use gemini-3.1-flash-lite for both director and triage
  • Experiment: Test different models to see which works best for your use case

Available Models

  • gemini-3.5-flash — Fast, balanced. Best default.
  • gemini-2.0-pro — Strongest. Slower, higher cost.
  • gemini-3.1-flash-lite — Cheapest. Good for triage (repo analysis).
  • Check Google's model list for latest options.

Set Director Model

For storyboard generation (the most important model):

export AUTOINTRO_DIRECTOR_MODEL=gemini-2.0-pro
npm run dev -- /path/to/repo

Set Triage Model

For repo analysis (reading files, detecting type):

export AUTOINTRO_TRIAGE_MODEL=gemini-3.1-flash-lite
npm run dev -- /path/to/repo

Set Both

export AUTOINTRO_DIRECTOR_MODEL=gemini-2.0-pro
export AUTOINTRO_TRIAGE_MODEL=gemini-3.1-flash-lite
npm run dev -- /path/to/repo

Or in .env:

AUTOINTRO_DIRECTOR_MODEL=gemini-2.0-pro
AUTOINTRO_TRIAGE_MODEL=gemini-3.1-flash-lite

Video Settings

Resolution & Frame Rate

Standard 1080p (Default)

AUTOINTRO_WIDTH=1920
AUTOINTRO_HEIGHT=1080
AUTOINTRO_FPS=30

Good balance of quality and render speed.

High-Resolution 1440p

AUTOINTRO_WIDTH=2560
AUTOINTRO_HEIGHT=1440
AUTOINTRO_FPS=30

Crisp, professional look. Takes ~1.5x longer to render.

YouTube 720p (Faster)

AUTOINTRO_WIDTH=1280
AUTOINTRO_HEIGHT=720
AUTOINTRO_FPS=30

Smaller file, faster rendering. Good for quick previews.

Smooth 60fps

AUTOINTRO_FPS=60

Doubles render time but animations look silky smooth.

Color Customization

Override the brand colors (from your code) with custom ones:

AUTOINTRO_ACCENT=#FF5733
AUTOINTRO_ACCENT2=#FFC300

Color values must be valid hex codes (6 digits + #).

Pro tip: Colors are auto-extracted from your project's CSS/Tailwind. Override only if you want a different look.

Error: "503 Service Unavailable" or "Rate Limited"

Why it happens: Gemini API is temporarily overloaded or you hit your usage quota.

Solution

1

Wait & Retry

AutoIntro auto-retries with exponential backoff. Usually resolves in 30 seconds.

If it still fails after 2–3 minutes:

2

Check Your Quota

Visit https://aistudio.google.com/app/apikey and check your usage.

Free tier: 15 requests per minute, generous daily limits.

3

Try a Cheaper Model

Swap to a lighter model to reduce load:

export AUTOINTRO_DIRECTOR_MODEL=gemini-3.1-flash-lite
export AUTOINTRO_TRIAGE_MODEL=gemini-3.1-flash-lite
npm run dev -- /path/to/repo
4

Try Again Later

If the API is globally overloaded, wait 5–10 minutes and retry.

Paid tier: If you consistently hit limits, consider upgrading to a paid Gemini plan for higher quotas.

Free Up Space

npm run dev -- /path/to/repo

Troubleshooting: Slow First Run

Problem: Takes 5–10 minutes on first run

Why it happens: On the first render, Remotion auto-downloads a headless Chromium (~100 MB). This is a one-time download.

What to Expect

  • First run: 5–10 minutes (includes Chromium download)
  • Subsequent runs: 2–5 minutes

Speed It Up

Reduce resolution/FPS:

export AUTOINTRO_WIDTH=1280
export AUTOINTRO_HEIGHT=720
export AUTOINTRO_FPS=24
npm run dev -- /path/to/repo

This renders faster while you're testing.

Is Chromium Stuck?

If it seems frozen, check your internet (Chromium download needs bandwidth). Open another terminal and watch the download:

tail -f ~/.cache/remotion/browser-download-log.txt
After the first run, it's much faster. Subsequent renders of the same repo take ~2 minutes.

Troubleshooting: Video Quality Issues

Problem: Trailer looks wrong, missing scenes, or off-brand

Quick Fixes

1. Review the Brief

Open <repo>/.auto-intro/brief.md and check if the narrative is correct.

If it's wrong: Use the web UI to edit the brief before re-rendering. The brief is the "contract" — if it's inaccurate, the video won't match your vision.

2. Re-render Without Re-analyzing

Click the "Re-render" button in the web UI. This reuses the analysis but regenerates the video (faster than a full rerun).

3. Edit the Storyboard (Power Users)

Advanced users can hand-edit <repo>/.auto-intro/storyboard.json to tweak scenes, timing, and animations. After editing, click "Re-render".

4. Use a Stronger Model

Swap to gemini-2.0-pro for higher-quality storyboard generation:

export AUTOINTRO_DIRECTOR_MODEL=gemini-2.0-pro
npm run dev -- /path/to/repo

5. Delete & Regenerate

Remove the .auto-intro/ folder and start fresh (clears cached analysis):

rm -rf /path/to/repo/.auto-intro
npm run dev -- /path/to/repo

If It Still Looks Bad

Possible causes:

  • Bad Gemini key: Check your API key is valid (visit aistudio.google.com/app/apikey)
  • Low quota: You may be out of free requests. Wait or upgrade.
  • Project is complex: AutoIntro works best on well-structured projects. Messy codebases may produce lower-quality results.

How AutoIntro Works (Technical)

The Five-Stage Pipeline

Stage 1: Analyze

Input: Path to a repo

Process: Read package.json, scan file tree, detect usage type (CLI, SDK, web app, API, mobile, desktop) using heuristics and dependency names.

Output: repo-facts.json (name, type, deps, complexity)

Stage 2: Inspect Design & UI

Input: Repo root

Process: Extract design token from source (Tailwind colors, CSS variables, component files). Detect brand logo.

Output: design-profile.json (palette, fonts, theme), component-inventory.json (UI components mapped to primitives)

Stage 3: Build Brief

Input: Repo facts + design

Process: Synthesize a concise narrative: project name, value prop, key features, usage flow, CTA. Approval Gate Here — you review before rendering.

Output: brief.json + brief.md

Stage 4: Direct (Gemini)

Input: Approved brief + design + inventory

Process: Gemini generates a scene-by-scene storyboard using the usage-type flow template + 42 animated UI primitives. Injects brand colors + typography.

Output: storyboard.json (scenes, timing, animations)

Stage 5: Render (Remotion)

Input: Storyboard

Process: Remotion spawns a headless Chromium, renders each frame, encodes to MP4. Wrapped in a crash-safe boundary.

Output: trailer.mp4

Privacy & Security

  • Local-first: Everything runs on your machine. No uploads (except to Gemini for analysis).
  • Gemini calls: Only code text + metadata leaves your device (repo facts, design profile, brief). Screenshots/video never uploaded.
  • Outputs: All artifacts stay in <repo>/.auto-intro/. You control where they go.

Crash Recovery

If rendering crashes, artifacts are preserved. You can re-run the same repo and AutoIntro reuses cached analysis + brief (much faster).

Editing the Brief

Why Edit?

The brief is the "contract" between analysis and rendering. If AutoIntro got the narrative wrong, edit it to set the story straight. The video will match your edited brief.

How to Edit (Via Web UI)

1

After Analysis

The web UI shows the brief in a markdown editor. Click the "Edit" tab to switch to edit mode.

2

Make Changes

Edit any field:

  • Project Name — what it's called
  • Usage Type — how people use it
  • Core Value — one-liner hook
  • Key Features — bullet list
  • Usage Flow — step-by-step
  • Call-to-Action — where users should go next
3

Save

Click "Save Changes". The brief updates in brief.json.

4

Re-render

Click "Approve & Generate" (or "Re-render" if you already rendered). Gemini will use your updated brief.

How to Edit (Direct File)

For power users, edit <repo>/.auto-intro/brief.json directly:

{
  "projectName": "MyTool",
  "usageType": "cli",
  "coreValue": "A fast CLI for X",
  "keyFeatures": ["Feature 1", "Feature 2"],
  "usageFlow": ["Step 1", "Step 2"],
  "callToAction": "Get Started"
}

Then re-render (web UI detects changes).

What Changes Affect the Video?

After editing the brief:

  • Re-render via web UI → Full regeneration (Gemini + Remotion) — takes 2–5 min
  • Or edit storyboard + click "Re-render" → Just Remotion (no Gemini) — takes ~2 min
If you only change colors/models, you can re-render the storyboard without changing the brief. If you change the narrative, re-run the full pipeline.

Editing the Storyboard (Power Users)

What's a Storyboard?

The storyboard is a JSON file that describes the video scene-by-scene: scene descriptions, timing, animations, text, colors. Remotion reads it and renders the actual video.

File: <repo>/.auto-intro/storyboard.json

Structure

{
  "scenes": [
    {
      "id": "scene-1",
      "type": "intro",
      "duration": 3,
      "title": "MyTool",
      "description": "A fast CLI tool",
      "animations": [...]
    },
    ...
  ]
}

When to Edit

  • Timing feels off (scenes too fast/slow) → adjust duration
  • Animations are janky → modify animations array
  • Text is wrong → edit title / description
  • Colors need tweaking → adjust colors in animation objects

How to Edit

1

Open the File

cat /path/to/repo/.auto-intro/storyboard.json
2

Make Changes

Edit JSON carefully (valid JSON is required). Use a JSON editor if possible.

3

Save & Re-render

In the web UI, click "Re-render". AutoIntro reads your edited storyboard and renders a new video (no Gemini call, ~2 min).

Caution: Invalid JSON will cause rendering to fail. Always validate JSON syntax before saving. Use a tool like JSONLint.

Quick Edits

Speed Up a Scene

"duration": 3  // Change to 2 for faster

Change Scene Title

"title": "New Title Here"

Modify Colors

Find hex color codes in animations and swap them:

"color": "#41A3EF"  // Change to your color

Frequently Asked Questions

Q: Does my code get uploaded?

No. Only text analysis (repo facts, design profile, narrative) is sent to Gemini. Video and screenshots never leave your machine.

Q: What if I don't have a Gemini key?

AutoIntro uses a lower-quality offline fallback. For best results, get a free key at https://aistudio.google.com/apikey. Takes 2 minutes.

Q: How long does it take to generate a video?

Analysis + render: 2–5 minutes on typical hardware (plus Chromium download on first run). Headless mode (--yes) is fastest.

Q: Can I use the videos commercially?

Yes. The video is yours to use. However, Remotion (the rendering engine) has a license requirement for companies >3 employees. See https://www.remotion.dev/license.

Q: Can I edit the video after?

The `.mp4` is final and playable in any video editor. For tweaks, re-render from AutoIntro (edit brief or storyboard + re-render).

Q: What project types work best?

Well-structured projects with clear UI, dependencies, and usage flows. CLI tools, SDKs, web apps, APIs, and mobile/desktop apps. Messy codebases may produce lower-quality results.

Q: Can I use it in CI/CD?

Yes. Use the `--yes` flag for headless, one-shot rendering. Useful for auto-generating trailers on release.

Q: Does it require Playwright?

No. Remotion uses its own headless Chromium. Playwright is not needed or used.

Q: What if Gemini rate-limits me?

AutoIntro auto-retries with exponential backoff. Free tier is 15 requests/min. If you consistently hit limits, upgrade your Gemini plan.

Q: Can I generate multiple trailers?

Yes. Run AutoIntro on different repos, or re-render the same repo with different briefs/settings. Each repo gets its own `.auto-intro/` folder.

Q: How do I report a bug?

Open an issue on GitHub: https://github.com/lehuubao/auto-intro/issues

Q: Is there a GUI for editing without JSON?

The web UI lets you edit the brief visually. For storyboard editing, you'll need to hand-edit JSON (power users only).
